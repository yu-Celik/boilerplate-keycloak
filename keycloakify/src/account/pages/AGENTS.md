# account/pages

## Purpose
Keycloak account console pages for user self-service â€” profile, security, applications, linked accounts, sessions, organizations, and audit log.

## Key Files

| File | Description |
|------|-------------|
| `PersonalInfo.tsx` | User profile (account.ftl). Shows first/last name, email, custom attributes. Form submit to update profile. Displays success/error flash messages. |
| `SigningIn.tsx` | Password management (password.ftl). Change password form. Show password strength requirements. Option to view password change history. |
| `Totp.tsx` | Two-factor authentication / authenticator app (totp.ftl). List configured authenticators, add new, remove/disable. Show QR code and shared secret for app setup. Backup codes management. |
| `Applications.tsx` | Connected OAuth applications (applications.ftl). Shows apps with granted permissions (scopes). Buttons to revoke access. Lists last access date. |
| `DeviceActivity.tsx` | Active sessions / device list (sessions.ftl). Show all active sessions with IP, device, browser, location. Option to sign out specific sessions. Shows last activity timestamp. |
| `Groups.tsx` | Groups and organization membership (groups.ftl). List organizations user is member of. Show role in each org (`/Admin`, `/Managers`, `/Members`). Links to leave org. |
| `LinkedAccounts.tsx` | Linked social / IdP accounts (federatedIdentity.ftl). Show social logins (Google, GitHub, etc.). Link/unlink social accounts to primary account. |
| `Log.tsx` | Security audit log (log.ftl). Display login history with timestamps, IP, browser, success/failure. Searchable, sortable event log. |

## For AI Agents

### Working In This Directory
- All pages receive `kcContext`, `i18n`, `Template`, `doUseDefaultCss` props
- Each page renders inside `<Template>` with `active` prop for sidebar highlighting
- **DO NOT use kcClsx()** â€” use Tailwind classes only
- Forms auto-generated by Keycloak; preserve all hidden inputs and attributes

### Page-to-pageId Mapping
See parent AGENTS.md for FTL template names and kcContext.pageId values.

### Form Pattern
```typescript
// All account pages follow this:
export default function PersonalInfo(props: AccountPageProps<...>) {
  const { kcContext, i18n, Template } = props;
  const { profile } = kcContext;

  return (
    <Template kcContext={kcContext} i18n={i18n} active="account">
      <form id={profile.attributes.form_id} action={profile.action} method="post">
        <input type="text" name="firstName" defaultValue={profile.attributes.firstName} />
        {/* CSRF token auto-inserted */}
        <button type="submit">{msgStr("doSave")}</button>
      </form>
    </Template>
  );
}
```

### Table Components
- Use shadcn/ui table component for device list, app list, log events
- Show pagination if many rows
- Sortable columns for log (timestamp, event, IP)
- Delete/revoke buttons for each row

### Styling Patterns
- **Active items**: `bg-primary/15 border border-primary/20`
- **Lists**: Use table component with row actions
- **Alerts**: Color-coded by severity (success green, error red, info blue)
- **Forms**: Input + Label shadcn components
- **Buttons**: Primary (submit), secondary (cancel), destructive (revoke/delete)

### Modal Dialogs
- Some actions show confirmation (e.g., "Revoke access?")
- Use Radix Dialog or AlertDialog shadcn components
- Prevent accidental data loss

### Internationalization
- All text via `msg()` or `msgStr()`
- Message keys are Keycloak standard (account, password, authenticator, etc.)
- No hardcoded English text

### Testing Requirements
- Build: `npm run build` in keycloakify/
- Restart: `docker compose restart keycloak`
- Access account console: http://localhost:3991/realms/boilerplate/account/
- Test each page: update profile, change password, add 2FA, check log, etc.
