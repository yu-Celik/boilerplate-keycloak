# account

## Purpose
Custom Keycloak account console theme with 8 pages for user account self-service: personal info, account security (passwords, 2FA), connected applications, linked social accounts, device sessions, groups/organizations, and activity audit log. All pages preserve Keycloak's form protocol while using Tailwind v4 and shadcn/ui.

## Key Files

| File | Description |
|------|-------------|
| `KcPage.tsx` | Router component — switches on `kcContext.pageId` to render the appropriate account page. Sets `doUseDefaultCss=false` for all pages. Falls back to `DefaultPage` for unknown pages. |
| `Template.tsx` | Layout shell — two-column layout with sidebar nav (account, password, 2FA, social, sessions, applications, groups, log) and main content area. Dark mode glassmorphism background with language switcher and logout button. |
| `i18n.ts` | Internationalization setup — wraps keycloakify's account `useI18n()` hook, provides message functions and language configuration. |
| `AccountApp.tsx` | (Typically unused; for testing/development reference) |

## For AI Agents

### Working In This Directory
- **Account pages are keycloakify components** — they receive `kcContext`, `i18n`, `Template`, `doUseDefaultCss` props
- **DO NOT use kcClsx()** — PatternFly is disabled; style with Tailwind only
- **Preserve form protocol** — forms require CSRF tokens and hidden inputs auto-managed by keycloakify; never remove them
- **Template sidebar nav** — built from `KcContext` features and URL objects (account, password, social, sessions, etc.); conditionally rendered based on `features` object
- **Message keys** — all UI text via `msg()` or `msgStr()` from i18n

### Template Component
**Header** (top bar):
- Branding icon + "Account" title
- Language switcher (conditionally shown if `enabledLanguages.length > 1`)
- "Back to app" link (from `referrer`)
- Sign out button

**Sidebar Navigation** (left, hidden on mobile):
- 8 nav items conditionally shown based on `features` object
- Current active item highlighted with `bg-primary/15`
- Icons defined inline (SVG)
- Uses `active` prop to detect current page

**Mobile Navigation** (horizontal bar, visible on small screens):
- Same nav items as sidebar, rendered as horizontal scrollable tab bar

**Main Content Area**:
- Scrollable content region with max-width constraint
- Flash message display (success, warning, error, info) with alert styling
- Page-specific form or content below message

### Form Handling Pattern
```typescript
// Example: PersonalInfo.tsx (account form)
export default function PersonalInfo(props: AccountPageProps<...>) {
  const { kcContext, i18n, Template } = props;
  const { profile } = kcContext;

  return (
    <Template kcContext={kcContext} i18n={i18n} active="account">
      <form id={profile.attributes.form_id} action={profile.action} method="post">
        <input type="text" name="firstName" defaultValue={profile.attributes.firstName} />
        {/* CSRF token auto-inserted by keycloakify */}
        <button type="submit">{msgStr("doSave")}</button>
      </form>
    </Template>
  );
}
```
- Form structure is auto-generated by keycloakify; preserve all form attributes
- Hidden inputs (CSRF, state) are auto-managed
- Submit button always `type="submit"`

### Styling Patterns
- **Active nav**: `bg-primary/15 text-primary border border-primary/20`
- **Inactive nav**: `text-muted-foreground hover:bg-accent/40 border border-transparent`
- **Flash messages**: colored alert borders/text (green for success, red for error, etc.)
- **Form inputs**: shadcn Input + Label components
- **Modal-like forms**: Card component with padding
- **Responsive**: Tailwind `hidden md:block` (hide sidebar on mobile), `md:hidden` (show mobile nav only)

### Internationalization
- `msgStr("account")` — get translated "Account" label
- `msg("backTo", referrer.name)` — interpolate referrer name
- Message keys are Keycloak standard account console keys (account, password, authenticator, etc.)

### Features Object
Keycloak account console exposes feature flags:
```typescript
features.passwordUpdateSupported   // bool
features.identityFederation        // bool (social login linked accounts)
features.log                       // bool (activity log shown)
// ... others
```
Use these to conditionally show/hide nav items and page sections
