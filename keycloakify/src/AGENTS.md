# src

## Purpose
Custom Keycloak theme source code (React 19, Tailwind v4, shadcn/ui). Dual theme system: login theme (14 pages for authentication flows) and account console theme (8 pages for user account management). Both themes use `doUseDefaultCss=false` to disable PatternFly and render with Tailwind directly.

## Key Files

| File | Description |
|------|-------------|
| `main.tsx` | Entry point — routes to login or account theme based on `window.kcContext.themeType`. Lazy-loads KcLoginPage and KcAccountPage with Suspense fallback. |
| `kc.gen.tsx` | Auto-generated by keycloakify build plugin — exports KcPage, KcContext types, and environment variable names. Do not edit manually. |

## For AI Agents

### Working In This Directory
- **Keycloakify v11**: no PatternFly — use Tailwind directly with `doUseDefaultCss=false`
- **Never use `kcClsx()`** — it depends on PatternFly CSS which is disabled
- **All form elements must preserve KC protocol**: hidden `name` attributes, form `action` values, CSRF token inputs — these are generated by keycloakify components and must not be removed
- **Dark mode**: `html` element has `dark` class set via `useSetClassName()` in both themes
- **Import shadcn components**: use `@/components/ui/` alias (configured in keycloakify tsconfig)
- **i18n via keycloakify**: `useI18n()` hook provides `msg()` and `msgStr()` for message keys; both themes define custom `i18n.ts`

### Theme Architecture
```
main.tsx
  → kcContext.themeType === "login"? → KcLoginPage (login/KcPage.tsx)
  → kcContext.themeType === "account"? → KcAccountPage (account/KcPage.tsx)

Each theme:
  → Routes to custom page component based on kcContext.pageId
  → Falls back to DefaultPage from keycloakify for unknown pages
  → Wraps content with Template (layout shell)
  → Uses i18n for all UI strings
```

### Build & Deploy
- `npm run build` in keycloakify/ — bundles with Vite, runs keycloakify plugin, generates `kc.gen.tsx`
- Docker build copies `keycloakify/dist/` → Keycloak theme directory
- Theme is live-reloaded in dev; production requires `docker compose restart keycloak`

### Common Patterns
- **Message rendering**: `msgStr("key")` for static strings, `msg("key", value)` for interpolation, `msg(..., { defaultMessage: "..." })` for fallback
- **Conditional nav items**: use `condition` field in item definition, filter at render time
- **Form handling**: all forms require CSRF token (auto-provided by keycloakify `useInitialize`), preserve all hidden inputs and form attributes
- **Icons**: inline SVG (no lucide-react in themes) to avoid bundle bloat
- **Responsive design**: Tailwind responsive prefixes (sm:, md:, etc.); mobile-first approach
