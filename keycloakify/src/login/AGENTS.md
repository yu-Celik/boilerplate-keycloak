# login

## Purpose
Custom Keycloak login theme with 14 pages covering the complete authentication flow: login, password, registration, password reset, email verification, profile update, password change, OTP, WebAuthn, organization selection, IdP profile review, terms, error pages, and consent. All pages preserve Keycloak's form protocol (CSRF, hidden inputs, name attributes) while styling with Tailwind v4 and shadcn/ui.

## Key Files

| File | Description |
|------|-------------|
| `KcPage.tsx` | Router component — switches on `kcContext.pageId` to render the appropriate page. Sets `doUseDefaultCss=false` for all pages. Falls back to `DefaultPage` for unknown pages. Uses lazy loading with Suspense. |
| `KcContext.ts` | TypeScript type extensions for the login theme context. Declares `KcContextExtension` (theme name, environment properties) and per-page extensions. |
| `i18n.ts` | Internationalization setup — wraps keycloakify's `useI18n()` hook, provides `msg()`, `msgStr()`, language list, and current language info. |
| `components/Template.tsx` | Layout shell — renders page header with branding, locale switcher, error messages; wraps form content with dark mode glassmorphism background |
| `components/Footer.tsx` | Optional footer component (auth links, terms, etc.) |

## For AI Agents

### Working In This Directory
- **All pages are keycloakify components** — they receive `kcContext`, `i18n`, `Template`, `classes`, `doUseDefaultCss` props
- **DO NOT use kcClsx()** — PatternFly is disabled; use Tailwind class strings directly
- **Preserve form protocol** — all form elements (inputs, buttons, hidden fields) are auto-generated by keycloakify; do not remove `name`, `id`, or hidden CSRF token inputs
- **Message keys** — use `msg("key")` or `msgStr("key")` for all UI text; keys are defined in Keycloak realm config or fallback to defaults
- **useI18n() hook** — get language, enabled languages, message functions from the i18n prop

- Renders dark mode glassmorphism header with locale switcher
- Wraps content in a centered card with radial gradient background glows
- Uses `useSetClassName()` to set `html.dark` class
- Uses `kcSanitize()` for user-provided content (XSS prevention)

### Form Handling Pattern
```typescript
// Example: LoginPassword.tsx
export default function LoginPassword(props: LoginPageProps<...>) {
  const { kcContext, i18n, Template } = props;
  const { form } = kcContext;

  return (
    <Template kcContext={kcContext} i18n={i18n}>
      <form id={form.id} onSubmit={() => true} action={form.action} method="post">
        {/* All form fields are auto-generated; preserve structure */}
        <input name="password" type="password" />
        {/* Hidden CSRF token is auto-inserted */}
        <button type="submit">{msgStr("doLogIn")}</button>
      </form>
    </Template>
  );
}
```
- Form `id`, `action`, `method` come from `kcContext.form` — do not modify
- All hidden inputs (CSRF, state tokens) are auto-managed by keycloakify
- Submit button is always `type="submit"` — binds to form submission

### Styling Patterns
- **Card layout**: use shadcn Card for form wrapper
- **Buttons**: use shadcn Button variants (primary, secondary, outline)
- **Inputs**: use shadcn Input component
- **Spacing**: Tailwind padding/margin utilities (p-6, space-y-4, etc.)
- **Text**: msg("key") strings only — never hardcode English text
- **Icons**: inline SVG, no lucide-react (keep bundle small)

### Internationalization
```typescript
const { msg, msgStr, currentLanguage, enabledLanguages } = useI18n({ kcContext });

msgStr("key")           // Synchronous string lookup (throws if missing)
msg("key")              // Async lookup with fallback
msg("key", { var: value })  // Interpolation: "Hello {var}"
msg("key", { defaultMessage: "..." })  // Fallback if key missing
```
