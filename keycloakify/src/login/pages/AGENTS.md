# login/pages

## Purpose
Custom Keycloak login theme pages — 13 page components covering the complete authentication flow with Tailwind v4 styling and shadcn/ui patterns.

## Key Files

| File | Description |
|------|-------------|
| `Login.tsx` | Initial login form (username/password). Maps to `login.ftl`. Form receives `kcContext.form` with CSRF token. Uses Template wrapper for layout. Handles both username+password flow and username-only flow. |
| `LoginUsername.tsx` | Username-only page (if configured). Maps to custom page ID. Captures username and submits to next step. |
| `LoginPassword.tsx` | Password challenge after username. Maps to `login-password.ftl`. Used in password-only or 2FA flows. Shows password input with optional "Forgot password" link. |
| `Register.tsx` | User registration form. Maps to `register.ftl`. Captures email, password, name fields. Includes password strength validation. Forms submission triggers email verification. |
| `LoginResetPassword.tsx` | Forgot password / password reset request. Maps to `login-reset-password.ftl`. Accepts email, sends reset link via KC. |
| `LoginVerifyEmail.tsx` | Email verification code entry. Maps to `login-verify-email.ftl`. User enters code sent to email. Simple form with minimal fields. |
| `LoginUpdateProfile.tsx` | Profile update form (first/last name, email). Maps to `login-update-profile.ftl`. Triggered on first login or profile completion requirement. |
| `LoginUpdatePassword.tsx` | Force password change. Maps to `login-update-password.ftl`. User required to change password before proceeding. |
| `LoginOtp.tsx` | One-time password (2FA) entry. Maps to `login-otp.ftl`. Shows OTP input field and backup code option. |
| `WebauthnAuthenticate.tsx` | WebAuthn/Passkey authentication. Maps to `webauthn-authenticate.ftl`. Shows "Authenticate with Passkey" button; uses browser WebAuthn API. |
| `SelectOrganization.tsx` | Organization selection during login. Maps to custom page ID. Multi-org users select initial org. Dropdown or card list UI. |
| `LoginIdpReviewProfile.tsx` | IdP profile review (social login). Maps to `idp-review-user-profile.ftl`. User reviews profile data from social provider before completing registration. |
| `Terms.tsx` | Terms of service acceptance. Maps to `terms.ftl`. Checkbox for ToS acceptance before proceeding. |
| `Error.tsx` | Generic error page. Maps to `error.ftl`. Shows error message and "Back to login" link. |
| `Consent.tsx` | OAuth consent screen. Maps to `consent.ftl`. Shows application permissions (scopes) and confirmation buttons. |

## For AI Agents

### Working In This Directory
- All pages receive `kcContext`, `i18n`, `Template`, `classes`, `doUseDefaultCss` props
- **DO NOT use kcClsx()** — PatternFly disabled; use Tailwind class strings directly
- **Preserve form protocol** — all form elements auto-generated by keycloakify; do NOT remove hidden CSRF inputs or form attributes
- **Message keys only** — use `msg("key")` or `msgStr("key")`; never hardcode English text
- Each page maps to a Keycloak FTL template name (shown in table above)

### Form Handling Pattern
```typescript
// All pages follow this pattern:
export default function LoginPage(props: LoginPageProps<...>) {
  const { kcContext, i18n, Template } = props;
  const { form } = kcContext;

  return (
    <Template kcContext={kcContext} i18n={i18n}>
      <form id={form.id} onSubmit={() => true} action={form.action} method="post">
        {/* Form fields auto-generated; preserve all attributes */}
        <input type="text" name="username" />
        {/* Hidden CSRF token auto-inserted */}
        <button type="submit">{msgStr("doLogIn")}</button>
      </form>
    </Template>
  );
}
```

### Styling Patterns
- **Cards**: shadcn Card wrapper with padding
- **Buttons**: shadcn Button (primary, secondary, outline variants)
- **Inputs**: shadcn Input + Label components
- **Spacing**: Tailwind spacing utilities (p-6, space-y-4, gap-4)
- **Typography**: Use Tailwind text utilities (text-sm, font-semibold)
- **Icons**: lucide-react only for UI chrome (no business logic icons)
- **Colors**: Use Tailwind color palette from theme-tokens.css

### Internationalization
```typescript
const { msg, msgStr, currentLanguage, enabledLanguages } = useI18n({ kcContext });

msgStr("key")           // Synchronous lookup (throws if missing)
msg("key")              // Async lookup
msg("key", { var: value })  // Interpolation
msg("key", { defaultMessage: "..." })  // Fallback
```

### Error Display
- Show `kcContext.formData?.['error']` if present
- Use shadcn Alert component with red variant
- Include error message from Keycloak (never suppress)

### Template Integration
- Pass `active` prop to Template for nav highlighting
- All pages must be wrapped in `<Template>` for consistent header/footer
- Template handles dark mode, locale switcher, layout shell
