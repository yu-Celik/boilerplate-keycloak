# components

## Purpose
Shared React components used across the dashboard: the sidebar shell, organization switcher, user nav, main nav, pending invitations banner, and a form submit button. Follows the shadcn/ui sidebar-07 pattern. The `ui/` subdirectory contains Radix UI primitive wrappers generated by the shadcn/ui CLI.

## Key Files

| File | Description |
|------|-------------|
| `app-sidebar.tsx` | Client component (`"use client"`) — composes `TeamSwitcher`, `NavMain`, `NavUser` inside shadcn `Sidebar`. Accepts `organizations`, `activeOrg`, and `user` props from the dashboard layout Server Component. |
| `team-switcher.tsx` | Client component — dropdown showing "Tous" (cross-org `__all__`), individual org list (by alias), and "Créer une organisation" link. Uses `useTransition()` for optimistic active-org update + `router.refresh()` after `switchOrg()` server action. Shows `Loader2` spinner during transition. |
| `nav-user.tsx` | Client component — sidebar footer dropdown with user avatar (initials fallback), "Mon compte" link to `/settings`, and "Déconnexion" button. Logout uses `fetch("/api/auth/logout", { method: "POST" })` then redirects to the returned `kcLogoutUrl` (OIDC end_session). Shows confirmation `AlertDialog` before logging out. |
| `nav-main.tsx` | Client component — renders the main nav items list using `SidebarMenuButton` with `asChild` and Next.js `Link`. Marks active item based on `usePathname()`. |
| `pending-invitations-banner.tsx` | Client component (`"use client"`) — renders a green banner above page content for each pending org invitation. Each invitation has a form submitting `acceptInvitation` server action. Returns `null` when `invitations.length === 0`. |
| `submit-button.tsx` | Client component — wraps `<button type="submit">` with `useFormStatus()` to show loading state during Server Action submission. |
| `logout-button.tsx` | Client component — standalone logout trigger (used outside of `NavUser` context). |

## For AI Agents

### Working In This Directory
- All components in this directory are `"use client"` (they use hooks or browser event handlers)
- `app-sidebar.tsx` is the composition root — it receives all data from the Server Component layout and distributes to child components via props (no client-side data fetching)
- Props flow: `(dashboard)/layout.tsx` (Server) → `AppSidebar` (Client) → `TeamSwitcher`, `NavMain`, `NavUser`

### TeamSwitcher Org-Switch Pattern
```typescript
function handleSwitch(alias: string) {
  setCurrentOrg(alias)                    // 1. Optimistic UI update (instant)
  startTransition(async () => {
    await switchOrg(alias)                // 2. Server Action sets httpOnly cookie
    router.refresh()                      // 3. RSC re-render with new org context
  })
}
```
- `useTransition()` keeps the UI responsive during the async server round-trip
- `isPending` state shows a `Loader2` spinner on the trigger button

### NavUser Logout Pattern
```typescript
// Client-side logout — must use fetch() not server action (needs redirect to KC)
const res = await fetch("/api/auth/logout", { method: "POST" })
const { kcLogoutUrl } = await res.json()
window.location.href = kcLogoutUrl  // Redirect to KC OIDC end_session_endpoint
```
- Uses `id_token_hint` via the API route to skip KC's logout confirmation screen

### Adding New Components
- Interactive components: add `"use client"` as the first line
- Pure display components receiving serializable props: can be Server Components (no directive needed)
- New shadcn/ui components: `npx shadcn@latest add <name>` — output goes to `ui/`
- Match existing pattern: Radix primitive + Tailwind utility classes, no inline styles
