# components

## Purpose
Shared layout components used across the dashboard: sidebar shell, organization/team switcher, user navigation menu, main navigation, pending invitations banner, and a form submit button wrapper. Follows shadcn/ui sidebar-07 pattern. The `ui/` subdirectory contains Radix UI primitive wrappers (generated by shadcn/ui CLI).

## Key Files

| File | Description |
|------|-------------|
| `app-sidebar.tsx` | Client component (`"use client"`) — composes TeamSwitcher, NavMain, NavUser inside shadcn Sidebar. Accepts `organizations`, `activeOrg`, `user` props from dashboard layout Server Component. |
| `team-switcher.tsx` | Client component — dropdown showing "Tous" (`__all__` cross-org), individual org list by alias, "Créer une organisation" link. Uses `useTransition()` for optimistic active-org update + `router.refresh()` after `switchOrg()` server action. |
| `nav-main.tsx` | Client component — renders main nav items list using SidebarMenuButton with `asChild` and Next.js Link. Marks active item based on `usePathname()`. |
| `nav-user.tsx` | Client component — sidebar footer dropdown with user avatar (initials fallback), "Mon compte" link to `/settings`, "Déconnexion" button. Logout uses `fetch("/api/auth/logout")` then redirects to KC OIDC `end_session_endpoint`. Shows AlertDialog confirmation before logout. |
| `pending-invitations-banner.tsx` | Client component — renders green banner above page content for each pending org invitation. Each invitation has form submitting `acceptInvitation` server action. Returns `null` when empty. |
| `submit-button.tsx` | Client component — wraps `<button type="submit">` with `useFormStatus()` to show loading state during Server Action submission. |
| `theme-toggle.tsx` | Client component — theme switcher (dark/light mode toggle). |

## For AI Agents

### Working In This Directory
- All components in this directory are `"use client"` (they use hooks or browser event handlers)
- `app-sidebar.tsx` is the composition root — receives all data from Server Component layout and distributes via props
- Props flow: `(dashboard)/layout.tsx` (Server) → `AppSidebar` (Client) → `TeamSwitcher`, `NavMain`, `NavUser`
- New shadcn/ui components: `npx shadcn@latest add <name>` generates into `ui/` subdirectory

### Adding New Components
- Interactive components: add `"use client"` as the first line
- Pure display components receiving serializable props: can be Server Components (no directive)
- Match existing pattern: Radix primitive + Tailwind utility classes, no inline styles
